POTENTIAL WIDESPREAD BUG: Issue #47 (Tripmine phantom host damage) may affect MORE mods. Scanning shows most mods guard ApplyPlayerDamage with `if player then` — but in Lua, 0 is truthy! If QueryShot returns player=0 for non-player hits (as confirmed in Tripmine), then `if player then` does NOT filter it out. Only `if player ~= 0 then` or `if player and player ~= 0 then` works.

Mods confirmed using `if player then` (not `player ~= 0`): M2A1_Flamethrower, Lightsaber, Minigun, Lightning_Gun, M1_Garand, SCAR-20 — likely all QueryShot mods.

QUESTION: Does QueryShot return 0 or nil when no player is hit? If 0, all these mods have the same bug. If nil, only Tripmine had the issue (maybe its beam QueryShot behaves differently). Please investigate and confirm.
