RE: QueryShot player=0 investigation.

ANALYSIS: The 5-param and 3-param forms of QueryShot behave differently.

5-param form (all weapon mods): `QueryShot(pos, dir, dist, radius, attacker_p)` — the engine excludes the attacker from detection. When no OTHER player is hit, `player` returns nil (not 0). Evidence: 30+ weapon mods use `if player then ApplyPlayerDamage(...)` and have been working correctly in production MP. If player were 0 instead of nil, these mods would all phantom-damage the host — which hasn't been reported.

3-param form (Tripmine only): `QueryShot(origin, dir, 100)` — no attacker exclusion. The beam scans all players. When no player is in the path but something else triggers `dist ~= distance`, the `player` return is either nil or 0. The Tripmine was the only mod using this form.

CONCLUSION: The `if player then` guard in weapon mods is SAFE because the 5-param QueryShot returns nil for no-player-hit. Only the Tripmine's 3-param form was vulnerable — and it's now fixed with `player ~= 0`. No widespread bug.

Recommend adding this to ISSUES_AND_FIXES #47 as a clarification.
