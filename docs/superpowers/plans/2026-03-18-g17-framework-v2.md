# G17 Framework Gun v2 Conversion Plan

> **For agentic workers:** This plan covers the G17 pistol AND establishes the conversion template for 8+ other framework guns by the same author (Tesseractahedron). All use identical code structure.
>
> **CRITICAL RULES for Teardown v2 mod code:**
> - `Players()`/`PlayersAdded()`/`PlayersRemoved()` are ITERATORS — NO `ipairs()`
> - Raw keys (`"rmb"`, `"r"`, etc.) do NOT take player param — use `InputPressed("rmb")` + ServerCall
> - Player param is ALWAYS LAST in Set*/Get* functions
> - ALWAYS run `python -m tools.lint --mod "ModName"` after writing any mod code

**Goal:** Convert the G17 (2638 lines v1) to v2 multiplayer, replacing the custom projectile system with v2 `Shoot()` API. This becomes the template for Kriss Vector, G36K, AK-12, SCAR, AK-105, AK-74, Saiga-12, Dragunov SVU.

**Architecture:** Replace custom projectile ballistics with v2 `Shoot()`. Server handles: tool registration, shooting, ammo, fire mode, reload logic, grenade explosions. Client handles: all input (raw keys), ADS/recoil/reload/melee animations, muzzle flash/impact particles, sounds, crosshair HUD, attachment menu UI, weapon sway.

**Key Design Decision:** The v1 custom projectile system (gravity, drag, penetration, ricochets, armor types — 250+ lines) is replaced with `Shoot(pos, dir, "bullet", damage, range, p, "g17")`. Rationale:
- v2 Shoot() auto-replicates via deterministic pipeline
- Provides kill attribution (kill feed) for free
- Handles MakeHole/damage automatically
- Loss of gravity/ricochet is negligible for pistol combat ranges
- Dramatically simplifies the conversion pattern for all framework guns

## Server/Client Split Summary

### Server (host only)
- `server.init()` — RegisterTool, SetToolAmmoPickupAmount, ammo.display
- `server.tick()` — PlayersAdded/Removed, fire mode tracking
- `server.tickPlayer()` — shoot timer, reload timer management
- `server.onShoot(p, px,py,pz, dx,dy,dz)` — Shoot() via v2 API, ammo decrement
- `server.onReload(p)` — reload state, ammo refill
- `server.onSelectFire(p)` — toggle fire mode
- `server.onShootGrenade(p, px,py,pz, dx,dy,dz)` — Explosion()
- `server.setOptionsOpen(p, open)` — options guard

### Client (each player)
- `client.init()` — LoadSound/LoadSprite/LoadLoop
- `client.tick()` — PlayersAdded/Removed
- `client.tickPlayer(p, dt)` — IsPlayerLocal input reading, all animation state machines
- Input reading: 15+ raw keys → ServerCall for actions, local state for animations
- ADS/recoil/reload/melee/inspect animations — SetToolTransform, SetShapeLocalTransform
- Muzzle flash + impact particles — SpawnParticle, PointLight
- Sounds — PlaySound
- Weapon sway — camerax/cameray (NOT mousedx/mousedy)
- `client.draw()` — crosshair, ammo display, fire mode indicator, keybind hints, attachment menu

### State Sync
- Ammo: registry sync `SetInt("g17."..p..".ammo", ammo, true)`
- Fire mode: registry sync `SetInt("g17."..p..".firemode", mode, true)`
- Reload state: registry sync `SetBool("g17."..p..".reloading", true, true)`

## Template Pattern for Other Framework Guns

To convert another Tesseractahedron framework gun (e.g., Kriss Vector):

1. Copy converted G17 as starting point
2. Change: tool ID, tool name, vox path, group number
3. Change: damage, velocity, recoil values, magazine sizes
4. Change: sound files
5. Change: ADS positions for different sights
6. Change: weapon-specific features (burst mode, etc.)
7. Keep: the entire server/client split architecture, input handling, animation system

Estimated per-gun conversion after G17 template: ~30 minutes (mostly find-replace + value tuning).
